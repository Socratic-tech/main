⚙ Setup & Connection

Connect to your Google Sheets backend to save and sync data across sessions.

⚠ URL must end in /exec — not /dev

📋 Setup Instructions
  1. Create a new Google Sheet.
  2. Go to Extensions → Apps Script and paste the contents of Code.gs. Save it.
  3. Select forceRebuild in the function dropdown and click ▶ Run. Accept permissions when prompted.
  4. Click Deploy → New Deployment → Web App.
  5. Set Execute as: Me and Who has access: Anyone. Click Deploy.
  6. Copy the URL — it should look like https://script.google.com/macros/s/ABC.../exec
  7. Paste it above and click Connect. You should see Connected ✓ in the top right.
🔧 Connection Troubleshooting
Still says "Not connected"? Work through this checklist:
  • In Apps Script: Deploy → Manage Deployments — confirm Who has access is Anyone (not "Anyone with Google account" or "Just me")
  • Make sure you copied the URL from Manage Deployments, not from the browser address bar
  • The URL must end in /exec — the /dev URL requires login and won't work here
  • If you edited Code.gs after deploying, you must redeploy with a New Version — the old deployment won't have the changes
  • Try opening the URL directly in a browser tab — you should see JSON data, not an error page
